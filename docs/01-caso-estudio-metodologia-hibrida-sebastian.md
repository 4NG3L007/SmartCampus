## Nombre de la metodologia

La metodologia hibrida elegida es **Scrum + Kanban**, tambien conocida como **Scrumban**.

Scrumban combina la organizacion por ciclos de trabajo de Scrum con la visualizacion continua de tareas de Kanban. Es util cuando un equipo necesita planificar avances, pero tambien necesita adaptarse rapidamente a cambios, retrasos o tareas urgentes.

## Por que se considera hibrida

Se considera una metodologia hibrida porque une elementos de dos metodologias agiles:

- De Scrum toma la planificacion por sprint, roles, reuniones y revision de avances.
- De Kanban toma el tablero visual, los estados de trabajo y el control del flujo de tareas.

En lugar de trabajar solo con reuniones y sprints, el equipo tambien puede ver claramente que tareas estan pendientes, en progreso, en pruebas o terminadas.

## Fases o pasos de la metodologia

### 1. Definicion del objetivo del sprint

El equipo define que quiere lograr en un periodo corto de trabajo. En el caso de SmartCampus, el objetivo del primer sprint es construir el flujo basico entre docente y estudiante.

### 2. Creacion del backlog

Se listan todas las tareas necesarias. Por ejemplo:

- Crear prototipo frontend.
- Definir base de datos.
- Definir endpoints.
- Registrar asistencia.
- Cargar notas.
- Probar la navegacion del sistema.

### 3. Priorizacion de tareas

El Product Owner o lider del equipo define que tareas son mas importantes para entregar primero. En SmartCampus, lo mas importante es mostrar el registro de asistencia y carga de notas.

### 4. Uso del tablero Kanban

Las tareas se organizan en columnas:

- Por hacer.
- En progreso.
- En pruebas.
- Terminado.

Esto ayuda a saber que esta haciendo cada integrante y evita que el equipo trabaje sin direccion.

### 5. Desarrollo de funcionalidades

Cada integrante trabaja en la funcionalidad asignada. En mi caso, desarrollo el frontend del sistema usando HTML, CSS y JavaScript.

### 6. Revision y pruebas

Se revisa si la funcionalidad cumple con lo esperado. Por ejemplo, se prueba si el usuario puede navegar entre docente, estudiante y dashboard.

### 7. Presentacion del avance

Cada integrante muestra su aporte, explica que hizo y defiende su parte del proyecto.

## Caracteristicas principales

- Permite organizar el trabajo por sprints.
- Usa un tablero visual para controlar tareas.
- Facilita el trabajo colaborativo.
- Ayuda a detectar bloqueos rapidamente.
- Es flexible cuando el equipo cambia o algun integrante no cumple.
- Permite entregar avances pequenos pero funcionales.

## Ventajas

- Es facil de entender y aplicar.
- Ayuda a distribuir tareas entre los integrantes.
- Permite ver el estado real del proyecto.
- No exige documentacion excesiva para empezar.
- Funciona bien en proyectos academicos con fechas cortas.
- Permite continuar aunque una parte del proyecto, como backend, todavia no este lista.

## Desventajas

- Si el equipo no actualiza el tablero, se pierde el control del avance.
- Si no hay lider claro, las tareas pueden quedar desordenadas.
- Puede confundirse con una simple lista de tareas si no se hacen revisiones.
- Requiere compromiso de cada integrante.
- No soluciona por si sola la falta de trabajo de algunos miembros.

## Proyecto donde la estoy usando

Estoy usando esta metodologia en el proyecto **SmartCampus**, una plataforma web de gestion academica. El proyecto necesita dividirse en frontend, backend, base de datos, pruebas y documentacion.

En mi caso, mi funcionalidad asignada es el **frontend del sistema**, especialmente:

- Panel de docente.
- Registro de asistencia.
- Carga de notas.
- Panel de estudiante.
- Dashboard basico con indicadores simulados.

La metodologia me sirve porque puedo avanzar con el frontend usando datos simulados mientras otros integrantes trabajan en backend, base de datos y pruebas.

## Recomendacion personal

Si recomiendo usar Scrumban para proyectos academicos y de desarrollo de software, especialmente cuando el equipo aun esta organizandose. Es una metodologia practica porque permite planificar, dividir tareas y visualizar avances de manera sencilla.

Tambien la recomiendo para SmartCampus porque el proyecto tiene varias partes que pueden desarrollarse en paralelo. Mientras una persona trabaja en frontend, otra puede trabajar en backend, otra en base de datos y otra en pruebas.

## Conclusion

Scrumban es una metodologia hibrida adecuada para SmartCampus porque combina planificacion, flexibilidad y seguimiento visual. Permite que cada integrante tenga una responsabilidad clara y facilita la entrega de avances individuales y grupales.
